# EchoLine 项目开发规范

## 语言与沟通
- 与用户沟通使用中文
- 代码注释使用中文
- Git commit message 使用英文，格式如下：
  - `feat: 新功能描述` — 新增功能
  - `fix: 修复描述` — 修复 bug
  - `chore: 维护描述` — 杂项（删文件、改配置等）
  - `refactor: 重构描述` — 重构代码（不改功能）
  - `style: 样式描述` — 纯样式调整

## 开发流程
- 复杂任务先进入 Plan Mode，方案确认后再动手
- 代码改完后，等用户测试确认后再提交
- 不要一次性改太多文件，保持每次提交的改动范围小而清晰

## 禁止事项
- 不要自作主张修改 UI 风格（颜色、字体、布局等），除非用户明确要求
- 不要添加用户没有要求的额外功能
- 不要删除或重命名用户没提到的文件
- 不要修改服务器配置（Nginx、PM2 等），除非用户明确要求

## 测试与调试
- 本地 videos/ 目录没有视频文件（太大不进 Git），本地无法测试视频播放
- 需要视频播放的功能，直接部署到服务器测试（用 Feature Flag 隔离）
- 新功能用 URL 参数 `?dev=1` 控制可见性，测试完毕再去掉开关
- 滚动字幕时用 `subtitleListEl.scrollTo()`，禁止用 `scrollIntoView`（会推动视频区域）
- 手机端测试必须做：按钮文字要加 `white-space: nowrap` 防止换行

## 项目部署
- 视频文件通过 scp 上传到服务器，不进 Git
- 字幕文件保留在本地 subtitles/ 目录，进 Git
- 服务器上的 episodes.json 修改后需同步回本地
